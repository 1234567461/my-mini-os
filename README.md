## 📖 第一阶段：引导扇区入门

### 🤔 电脑是怎么启动的？

按下电源键后，发生了什么？

```
1. 加电自检 (POST)
   ↓
2. BIOS/UEFI 初始化硬件
   ↓
3. BIOS 按顺序查找可引导设备（硬盘、U盘、光盘...）
   ↓
4. 找到引导扇区（512字节，最后两个字节是 0xaa55）
   ↓
5. 把引导扇区加载到内存 0x7c00 位置
   ↓
6. 跳转到 0x7c00，开始执行我们的代码！
   ↓
7. 我们的OS开始运行啦！
```

### 📝 关键知识点

**1. org 0x7c00**
```nasm
org 0x7c00
```
告诉汇编器：我们的代码会被加载到内存 `0x7c00` 这个位置。
为什么是 `0x7c00`？这是当年IBM PC定下的规矩，后来就成了行业标准。

**2. BIOS中断**
```nasm
int 0x10
```
BIOS提供了很多基础功能（显示字符、读磁盘、键盘输入等），我们通过"中断"来调用它们。
- `int 0x10` = 视频服务（显示文字、设置模式等）
- `int 0x13` = 磁盘服务
- `int 0x16` = 键盘服务

**3. 引导扇区魔数**
```nasm
dw 0xaa55
```
引导扇区的最后两个字节**必须**是 `0xaa55`。
BIOS会检查这个标志，只有看到它才会认为这是一个可引导的扇区。

> 💡 注意：x86是"小端序"，所以虽然我们写的是 `0xaa55`，但在文件里实际存储的是 `0x55` 在前，`0xaa` 在后。

## 📖 第二阶段：32位保护模式 ✨

### 🎯 本阶段实现功能

- ✅ 读取磁盘：把内核从磁盘读到内存
- ✅ 设置GDT：全局描述符表
- ✅ 切换模式：从16位实模式 → 32位保护模式
- ✅ 32位内核：直接操作VGA显存显示文字

### 🤔 什么是实模式和保护模式？

**实模式（Real Mode）**：
- 8086时代的老模式
- 只能访问 1MB 内存
- 没有内存保护，程序可以随便读写任何内存
- BIOS中断只能在实模式下用

**保护模式（Protected Mode）**：
- 80286开始引入的现代模式
- 可以访问 4GB 内存（32位）
- 有内存保护机制，程序不能随便搞破坏
- 有特权级（Ring 0 ~ Ring 3）
- 现代操作系统都运行在保护模式

### 📋 切换到保护模式的步骤

```
1. 关中断（cli）
   ↓
2. 加载GDT（lgdt）
   ↓
3. 设置CR0寄存器的PE位
   ↓
4. 远跳转（jmp CODE_SEG:xxx）
   ↓
5. 初始化所有数据段寄存器
   ↓
6. 设置栈
   ↓
7. 进入32位世界！
```

### 🏗️ 什么是GDT？

GDT = Global Descriptor Table（全局描述符表）

它是保护模式的"户口本"，定义了各个内存段的信息：
- 段基地址（从哪里开始）
- 段界限（有多大）
- 特权级（Ring 0 还是 Ring 3）
- 是代码段还是数据段
- 可读/可写/可执行权限

我们的GDT里有3个描述符：
1. **空描述符**：必须有，第一个都是空的
2. **代码段描述符**：放代码的地方，可读可执行
3. **数据段描述符**：放数据的地方，可读可写

### 💻 32位下怎么显示文字？

进入保护模式后，**BIOS中断不能用了**！我们要直接操作VGA文本缓冲区。

VGA文本缓冲区地址：`0xb8000`

每个字符占2个字节：
- 第1字节：ASCII字符
- 第2字节：属性（颜色）

```
颜色属性字节：
  7 6 5 4   3 2 1 0
  └─────┘   └─────┘
  背景色    前景色
```

颜色代码：
- 0 = 黑色
- 1 = 蓝色
- 2 = 绿色
- 3 = 青色
- 4 = 红色
- 5 = 品红
- 6 = 棕色
- 7 = 浅灰
- 8 = 深灰
- 9 = 亮蓝
- A = 亮绿
- B = 亮青
- C = 亮红
- D = 亮品红
- E = 黄色
- F = 白色

## 🧪 动手实验

试试改这些地方，看看效果：

1. **改文字颜色**：把 `kernel_entry.asm` 里的 `mov ah, 0x0f` 改成 `mov ah, 0x0c`（红色）
2. **改显示内容**：修改 `kernel_msg`、`mode_msg` 这些字符串
3. **试试删掉GDT**：看看会发生什么（大概率会崩溃）
4. **单步调试**：用 `make debug` + GDB 单步执行，看看每一步寄存器的变化
