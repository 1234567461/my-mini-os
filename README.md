# 🖥️ My Mini OS

从零开始开发的迷你操作系统，跟着代码一步步学习计算机底层原理！

---

## 📚 项目介绍

这是一个**从零开始手写的操作系统学习项目**，目标是从最底层的引导扇区开始，一步步构建一个能运行的迷你操作系统。

**适合人群**：
- 想彻底搞懂电脑是怎么启动的
- 对操作系统底层原理好奇
- 想挑战硬核编程
- 有一定C语言/汇编基础（没有也没关系，跟着学就行）

## 🛠️ 环境搭建

### 需要的工具

| 工具 | 作用 | 安装命令（Ubuntu/Debian） |
|------|------|--------------------------|
| nasm | 汇编编译器 | `sudo apt install nasm` |
| qemu-system-x86 | 虚拟机（运行我们的OS） | `sudo apt install qemu-system-x86` |
| make | 构建工具 | `sudo apt install make` |
| xxd | 查看二进制（可选） | 系统自带 |

### 一键安装（Ubuntu/Debian）

```bash
sudo apt update && sudo apt install -y nasm qemu-system-x86 make
```

### macOS

```bash
brew install nasm qemu make
```

### Windows

推荐用 **WSL2** (Windows Subsystem for Linux)，然后按Ubuntu的方式安装。

## 🚀 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/1234567461/my-mini-os.git
cd my-mini-os
```

### 2. 编译

```bash
make build
```

### 3. 运行！

```bash
make run
```

你会看到QEMU虚拟机启动，然后显示：
```
Hello, My Mini OS!

This is my first operating system!
```

🎉 **恭喜！你运行了自己的第一个操作系统！**

### 4. 退出QEMU

按 `Ctrl + A` 然后按 `X` 退出QEMU。

## 📖 第一阶段：引导扇区详解

### 🤔 电脑是怎么启动的？

按下电源键后，发生了什么？

```
1. 加电自检 (POST)
   ↓
2. BIOS/UEFI 初始化硬件
   ↓
3. BIOS 按顺序查找可引导设备（硬盘、U盘、光盘...）
   ↓
4. 找到引导扇区（512字节，最后两个字节是 0xaa55）
   ↓
5. 把引导扇区加载到内存 0x7c00 位置
   ↓
6. 跳转到 0x7c00，开始执行我们的代码！
   ↓
7. 我们的OS开始运行啦！
```

### 📝 引导扇区代码解析

文件位置：`src/boot/bootsect.asm`

这是一个完整的、只有512字节的"操作系统"。虽然只能显示文字，但它真的是一个可引导的程序！

#### 关键知识点：

**1. org 0x7c00**
```nasm
org 0x7c00
```
告诉汇编器：我们的代码会被加载到内存 `0x7c00` 这个位置。
为什么是 `0x7c00`？这是当年IBM PC定下的规矩，后来就成了行业标准。

**2. BIOS中断**
```nasm
int 0x10
```
BIOS提供了很多基础功能（显示字符、读磁盘、键盘输入等），我们通过"中断"来调用它们。
- `int 0x10` = 视频服务（显示文字、设置模式等）
- `int 0x13` = 磁盘服务
- `int 0x16` = 键盘服务

**3. 引导扇区魔数**
```nasm
dw 0xaa55
```
引导扇区的最后两个字节**必须**是 `0xaa55`。
BIOS会检查这个标志，只有看到它才会认为这是一个可引导的扇区。
> 💡 注意：x86是"小端序"，所以虽然我们写的是 `0xaa55`，但在文件里实际存储的是 `0x55` 在前，`0xaa` 在后。

你会看到：
- 文件大小正好是 512 字节
- 最后两个字节是 `55 aa`（小端序的 0xaa55）
- 前面是我们的代码和数据

## 💡 小提示

1. **不要怕看不懂**：操作系统开发涉及很多底层知识，一开始看不懂很正常。多看几遍，动手试试，慢慢就懂了。

2. **多动手实验**：改改代码，看看会发生什么。比如把字符串改了，把颜色改了，甚至故意写错看看会怎么报错。

3. **善用调试工具**：QEMU + GDB 可以单步调试汇编代码，看着寄存器一步步变化，理解会特别深刻。

4. **遇到问题先搜OSDev Wiki**：那上面几乎有你遇到的所有问题的答案。
